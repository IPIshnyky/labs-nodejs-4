# Документація скриптів заповнення БД (Seeding)

### Огляд
Цей розділ проєкту реалізує автоматизоване заповнення таблиці `tasks` тестовими даними чотирма різними способами. Це демонструє еволюцію моделей асинхронності в Node.js.

### Локальний запуск (Команди)
Щоб заповнити базу даних конкретним методом, виконайте відповідну команду в терміналі:

- **Sync-style:** `node scripts/seed-sync.js`
- **Callback:** `node scripts/seed-callback.js`
- **Promise:** `node scripts/seed-promise.js`
- **Async/Await:** `node scripts/seed-async.js`

### Як це працює
- **Джерело даних:** Усі записи зчитуються з файлу `data/tasks.json`.
- **Мапінг даних:** Текстові значення пріоритетів ("high", "medium", "low") автоматично конвертуються в цілі числа (3, 2, 1) відповідно до схеми бази даних.
- **Database Client:** Скрипти використовують спільний екземпляр `pool` із `src/db/index.js` та підключаються через змінну оточення `DATABASE_URL`.
- **Консистентність:** Незалежно від обраного стилю написання коду, усі скрипти заповнюють базу даних ідентичним набором даних.

### Обслуговування бази даних
- **Очищення:** Перед повторним тестуванням скриптів використовуйте наступну команду, щоб видалити дані та скинути лічильники ID:
`TRUNCATE tasks RESTART IDENTITY;`
- **Автоінкремент:** Поле `id` виключено з JSON-файлу, оскільки PostgreSQL обробляє його автоматично через первинний ключ типу `SERIAL`.

### Нотатки щодо реалізації
Кожен скрипт демонструє окремий підхід до обробки асинхронних операцій введення-виведення в Node.js: від традиційних колбеків і промісів до сучасного синтаксису async/await. Це забезпечує виконання критерію однакового заповнення бази даних у всіх версіях.
